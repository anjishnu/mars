\section{Conclusions and Future Work}
\label{sec:conclusion}

We described MARS, a single-binary terminal editor, multiplexer, and agent with memory
over the user's own local context. In the terminal's constrained domain, lexical
retrieval over local files raised a small model from 38\% to 96\% on personalized
translation and from 17\% to 93\% on self-reconfiguration. We read this as the first
rung of a ladder. Above
single-command recall lies \emph{skill learning}: mining repeated command sequences into
reusable procedures. Above that lies \emph{workflow understanding}: recognizing a
training run or debug session in a command stream, predicting the next step, and
flagging deviation. Furthest out is \emph{cross-fleet memory}: semantic retrieval once
paraphrase matters, and memory that follows the user across machines through the same
key broker. The through-line is ownership: local, private, portable, compounding
memory, which an assistant that sees only what it is handed cannot replicate.
A cloud assistant starts every conversation a stranger; an agent that owns its user's
memory becomes a little less generic, and a little less amnesiac, with every
correction. Simple memory works surprisingly well; the same substrate, enriched, points
to agents that learn your skills and understand your work.
