\section{Related Work}
\label{sec:related}

\paragraph{Terminal tooling.}
Multiplexers such as tmux \citep{tmux} and Zellij \citep{zellij} provide persistence but
no intelligence. Atuin \citep{atuin} makes the user's shell history a first-class
searchable database, and thefuck \citep{thefuck} repairs a mistyped command from its
error output; both share our premise that the user's own history is valuable, but
neither feeds it to a model. AI terminals such as Warp \citep{warp} add command search
but see only their own blocks. CLI agents such as Claude Code
\citep{anthropic2024claudecode} and GitHub Copilot in the CLI \citep{copilotcli} put a
cloud model in the shell and can read the same local files MARS reads; however, they run
per invocation, so nothing persists between calls, corrections are not captured as they
happen, and every remote machine needs its own credential. IDE agents such as Cursor
\citep{cursor} lose context when the window closes. MARS differs in the combination: one
persistent process that hosts the editor, captures every accepted correction, grounds
the agent in its own action registry, and keeps the credential home.

\paragraph{Natural language to commands.}
NL2Bash \citep{lin2018nl2bash} and Tellina \citep{lin2017tellina} established the
NL-to-shell task and its corpora; we build on the task rather than the models,
personalizing translation with the user's own history.

\paragraph{Agents and retrieval.}
Directive-style acting relates to ReAct \citep{yao2023react} and tool use to Toolformer
\citep{schick2023toolformer}, though we deliberately use a portable text protocol with
registry grounding rather than provider-native tool calls. Our memory is
retrieval-augmented generation \citep{lewis2020rag} in its simplest lexical form; the
novelty is the setting, retrieval over the user's \emph{own} local context, embedded
where it is produced.

\paragraph{Computer-use agents.}
A parallel line of work drives whole desktops rather than terminals. Anthropic's
computer use \citep{anthropic2024computeruse} operates GUIs from screenshots, OSWorld
\citep{xie2024osworld} benchmarks such agents on real operating systems, OS-Atlas
\citep{wu2025osatlas} pretrains a foundation action model for GUI grounding, and
Plan-and-Act \citep{erdogan2025planandact} separates a planner from an executor to
survive long-horizon tasks. The terminal offers the same ambitions a simpler substrate:
the action space is already text, so grounding is a registry lookup rather than a vision
problem, and a plan is a command sequence the user can read. These
techniques map onto the rungs above single-command recall
(\S\ref{sec:conclusion}): procedure mining is a planning problem, and workflow
understanding is the terminal's analogue of GUI-state tracking.
