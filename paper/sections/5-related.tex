\section{Related Work}
\label{sec:related}

\paragraph{Terminal tooling.}
Multiplexers such as tmux \citep{tmux} and Zellij \citep{zellij} provide persistence but
no intelligence. Atuin \citep{atuin} and thefuck \citep{thefuck} share our premise that
the user's own history is valuable, but neither feeds it to a model. AI terminals such
as Warp \citep{warp} add command search but see only the commands run inside them. CLI agents such as Claude Code
\citep{anthropic2024claudecode} and GitHub Copilot in the CLI \citep{copilotcli} put a
cloud model in the shell and can read the same local files MARS reads; however, at the
time of writing their persistent context is curated by hand (project memory files)
rather than captured automatically from the commands a user accepts, and every remote
machine needs its own credential; IDE agents \citep{cursor} lose context when the
window closes. MARS differs in the combination: one
persistent process that hosts the editor, captures every accepted correction, grounds
the agent in its own action registry, and keeps the credential home.

\paragraph{Agents, retrieval, and NL-to-commands.}
NL2Bash \citep{lin2018nl2bash} and Tellina \citep{lin2017tellina} established the
NL-to-shell task and its corpora; we build on the task rather than the models,
personalizing translation with the user's own history.
Directive-style acting relates to ReAct \citep{yao2023react} and tool use to Toolformer
\citep{schick2023toolformer}, though we use a portable text protocol with registry
grounding rather than provider-native tool calls. Our memory is
retrieval-augmented generation \citep{lewis2020rag} in its simplest lexical form; the
novelty is the setting: retrieval over the user's \emph{own} local context, embedded
where it is produced. The residual failures we observe on Axis B, where a strong
parametric prior overrides a retrieved specific (\S\ref{sec:discussion}), are a
deployed instance of the context-versus-parametric knowledge conflicts studied in
question answering \citep{longpre2021entity}; the personalization axis relates to
benchmarks such as LaMP \citep{salemi2024lamp}, though our memory is captured from use
rather than provided as a profile.

\paragraph{Computer-use agents.}
A parallel line of work drives whole desktops rather than terminals: Anthropic's
computer use \citep{anthropic2024computeruse} operates GUIs from screenshots, OSWorld
\citep{xie2024osworld} benchmarks such agents, OS-Atlas \citep{wu2025osatlas} pretrains
a GUI-grounding action model, and Plan-and-Act \citep{erdogan2025planandact} separates
planning from execution. The terminal offers those ambitions a simpler substrate:
the action space is already text, so grounding is a registry lookup rather than a vision
problem, and a plan is a command sequence the user can read.
