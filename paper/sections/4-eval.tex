\section{Evaluation}
\label{sec:eval}

\subsection{Setup}
\label{sec:setup}

We evaluate the two memory axes on frozen, a priori, author-written gold sets and a
single small model
under test: 112 command-translation items (88
general, 24 project-specific) and 82 self-knowledge question--answer items (52
knowledge, 30 reconfiguration); Axis A exercises Memory 1 (\S\ref{sec:m1}), Axis B
Memory 2 (\S\ref{sec:m2}). The model under test is \code{claude-haiku-4-5}: the
thesis requires a \emph{small} model, so that personalization has a gap to close, but
not necessarily a free one. Two free tiers
(Qwen3-32B on Groq, Gemini Flash-Lite) exhausted their daily quotas mid-batch (the
wall the cascade of \S\ref{sec:cascade} routes around), so we ran the final numbers
on a small paid model; all reported runs had zero empty completions.

Correctness is scored by a stronger judge model, \code{claude-sonnet-5}, following
LLM-as-judge practice \citep{zheng2023judging}, so that the model under test does not
grade itself; judge and testee share a model family (see
Limitations).\footnote{Both models ran at provider-default sampling (Claude 4.5-series
models reject the temperature parameter); prompt templates ship with the source
(\code{src/prompts/}), alongside the gold sets and frozen result files.} The
corrective-memory
store is seeded per condition from paraphrases generated by a separate LLM pass and
frozen alongside the gold sets; it contains the 24 seeded pairs
and no distractors, and retrieval injects the top three (\S\ref{sec:m1}). Prompts,
contexts, and token counts come from the self-instrumenting log of \S\ref{sec:system}.

\subsection{Axis A: corrective memory closes an information gap}
\label{sec:axisa}

\begin{table}[t]
\centering
\small
\begin{tabular}{@{}lc@{}}
\toprule
\textbf{Setting} & \textbf{Accuracy} \\
\midrule
General ($n{=}88$)        & 90\% (79/88) \\
Personalized ($n{=}24$)   & 42\% (10/24) \\
Overall ($n{=}112$)       & 79\% (89/112) \\
\midrule
\multicolumn{2}{@{}l}{\emph{Corrective memory ($n{=}24$, leakage-controlled)}} \\
\quad Cold (no memory)             & 38\% (9/24) \\
\quad Paraphrase memory            & \textbf{96\% (23/24)} \\
\quad Verbatim memory (ceiling)    & 100\% (24/24) \\
\bottomrule
\end{tabular}
\caption{Command translation (\code{claude-haiku-4-5}, judged by
\code{claude-sonnet-5}). The corrective-memory block is leakage-controlled
(\S\ref{sec:axisa}): 96\% measures generalization across a paraphrase gap; verbatim
is the lookup ceiling. Base and cold come from separate runs that disagree on five
items in both directions, netting to one (Appendix~\ref{app:examples}).}
\label{tab:axisA}
\end{table}

The base numbers in Table~\ref{tab:axisA} confirm the premise: the small model is a
competent general shell author at 90\% but falls to 42\% on project-specific commands
it cannot know. These are missing-knowledge failures, chiefly \emph{ecosystem
blindness} (Appendix~\ref{app:examples}), exactly what local memory targets.

The corrective-memory experiment isolates that fix under the leakage-controlled
protocol (stored paraphrases at mean word-level Jaccard 0.36 to the test query, tested
on the original requests). Cold accuracy is 38\% (9/24); paraphrase memory raises it
to 96\% (23/24), with fifteen items flipping to correct and one regressing from
correct to wrong (the appendix examines the flips and the regression); verbatim memory
reaches the lookup ceiling of 100\% (24/24): simple BM25 over a local file recovers
almost all of the personalization gap by generalizing, not by lookup. Memory adds
about 66 tokens per call; the sign of that cost is model-dependent: on a reasoning
model, the same exemplars suppress chain-of-thought and net cost falls
(Appendix~\ref{app:examples}).

\subsection{Axis B: docs memory grounds identifiers, not priors}
\label{sec:axisb}

\begin{table}[t]
\centering
\small
\begin{tabular}{@{}lcc@{}}
\toprule
\textbf{Question type} & \textbf{No memory} & \textbf{+\,Docs memory} \\
\midrule
Knowledge (how-do-I)   & 58\% (30/52) & 65\% (34/52) \\
Reconfigure            & 17\% (5/30)  & \textbf{93\% (28/30)} \\
Overall ($n{=}82$)     & 43\% (35/82) & \textbf{76\% (62/82)} \\
\bottomrule
\end{tabular}
\caption{Self-knowledge and self-reconfiguration (same models). Docs memory is
decisive on reconfiguration, where retrieved knob descriptions supply the exact file,
name, and default. The docs condition also adds an explain-don't-act instruction;
Appendix~\ref{app:examples} separates its effect from that of the retrieved
identifiers.}
\label{tab:axisB}
\end{table}

Retrieved documentation lifts accuracy on questions about MARS from 43\% to 76\%
(Table~\ref{tab:axisB}), but the gain is unevenly distributed. On ``how-do-I''
knowledge questions it is modest (58\% to 65\%): the no-memory model already answers
many correctly, and even with docs it sometimes hallucinates a keybinding the docs
state exactly (Appendix~\ref{app:examples}); \S\ref{sec:discussion} gives this
asymmetry a shape. The decisive gain is on \emph{reconfiguration} (turning a request
like ``autosave more often'' into a concrete knob edit), which rises from 17\% to
93\%. Without the docs the model invents config filenames; with the knob descriptions
retrieved, it emits the exact \code{knob = value} in \code{tuning.json}. One caveat:
the docs condition also adds an explain-don't-act instruction, which repairs answers
that fired a directive instead of explaining; the identifier-level repair requires the
retrieved text (Appendix~\ref{app:examples}).


\subsection{Discussion: when retrieval beats the prior}
\label{sec:discussion}

Read together, the two axes locate the boundary of lexical memory. Retrieval is
decisive when the missing piece is an identifier the model can copy intact: a project's
\code{cargo test}, a \code{-{}-config} path, a knob name carrying its file and default.
It is weakest when the model holds a strong parametric prior over the same slot:
keybindings gain little because a plausible chord such as \code{C-f} overwrites the
retrieved \code{C-s}, and the experiment's single regression (\code{p19}) is the same
conflict in miniature, a Rust-shaped ecosystem prior overriding a weakly retrieved
Python exemplar. This is a naturally occurring instance of the
context-versus-parametric knowledge conflicts studied in question answering
\citep{longpre2021entity}, observed here in a deployed agent rather than a constructed
benchmark. It yields a design rule: place strings the model must not paraphrase (flags,
paths, identifiers) into memory verbatim, and expect diminishing returns wherever the
answer competes with a common default; the next unit of complexity is
recency-and-project weighting on the retrieved side of these conflicts
(\S\ref{sec:future}).
