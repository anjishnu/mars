\section{Qualitative Examples and Error Analysis}
\label{app:examples}

Every example in this appendix is quoted verbatim from the frozen result files the
evaluation was scored on: \code{axis\_a.jsonl} (112 translation items),
\code{lift.jsonl} (24 leakage-controlled cold/paraphrase/verbatim items), and
\code{axis\_b.jsonl} (self-knowledge, 164 rows over the two memory conditions), with the
model under test \code{claude-haiku-4-5} and the judge \code{claude-sonnet-5}
throughout. The purpose is to show, from the raw data, exactly where a simple
local-memory mechanism helps, where it does not, and where the measurement itself is
the limiting factor.

On general commands the small model is a competent shell author (79 of 88 correct), and
its correct outputs are not only the easy one-liners: it composes multi-stage pipelines
with the right utility at each stage. Judged correct, for example:

\begin{footnotesize}
\begin{verbatim}
a20  find the ten largest files under the cwd
   -> find . -type f -printf '%s %p\n' \
        | sort -rn | head -10
\end{verbatim}
\end{footnotesize}

\noindent This is the premise of the paper made concrete: general shell translation is a
near-solved commodity even on a small, inexpensive model, so the accuracy that matters
is the personalized slice, which the model cannot know from the request text alone.

Two error classes account for the general-command failures (this run produced zero
empty completions, so every failure is a real answer). The first is \emph{placeholder
and precision misses}: a stand-in the gold set spells differently (\code{a69}
\code{cat -n} vs.\ gold \code{cat -n file.txt}) or a plausible but semantically
different tool (\code{a38} ``show my current IP'' $\rightarrow$ \code{ip addr show},
a LAN address, where the reference wants \code{curl ifconfig.me}). The second
dominates the personalized slice: \emph{ecosystem blindness}, answering a Rust or
\code{uv} project with the statistically commonest ecosystem, as in \code{p04} ``run
the unit tests'' $\rightarrow$ \code{npm test} (gold \code{cargo test});
Table~\ref{tab:lift-examples} quotes more. A milder pattern is \emph{honest
abstention}: on the hardest project commands the model declines rather than bluff,
safe behavior that still scores wrong against a gold command.

The judge assigns a \emph{partial} band that captures near-misses, and its slack
explains why the two no-memory measurements of the 24 personalized items disagree: the
base row of Table~\ref{tab:axisA} (42\%, 10/24) and the lift experiment's cold
condition (38\%, 9/24) come from separate runs, and they differ on five items. Three
differ because the cold completions were regenerated and the model produced a different
command (\code{p10}, \code{p14}, \code{p17}); two because the judge graded the
identical output differently across runs (\code{p01}
\code{source venv/bin/activate}, correct vs.\ partial; \code{p22} \code{docker push},
wrong vs.\ correct). The disagreements move in both directions and net to one item.
Every point estimate in this paper therefore carries a couple of points of
generation-and-judging slack, and we read the results through that band.

\subsection{Corrective memory}

Under the leakage-controlled protocol, with only a \emph{paraphrase} of each prior
request in the store (mean word-Jaccard 0.36 to the test query), 23 of 24
project-specific items are correct, up from 9 correct cold: 15 items flip to correct
and one regresses (below); Table~\ref{tab:lift-examples} quotes representative flips
verbatim. Two sub-mechanisms are visible: \emph{ecosystem correction} (the commonest
default corrected to the user's real toolchain) and \emph{exact-argument recall} (the
right utility gains the precise config, path, or split the user expects). Of the 15
non-correct cold items, 9 are outright wrong and 6 partial, so memory repairs both
fundamentally-wrong ecosystem guesses and last-mile precision misses.


The paraphrase column's single error is also the experiment's one \emph{regression}.
Cold, \code{p19} ``run the benchmark suite'' was answered \code{make benchmark},
which the judge accepted as a plausible route to the gold
\code{python -m benchmarks.run}. With the paraphrased exemplar retrieved, the model
instead answered \code{cargo bench}: it resolved the ecosystem to Rust, and the
retriever did not surface the paraphrased Python exemplar strongly enough to override
that prior. This is a retrieval-plus-prior miss, not a lookup failure, and it is the
honest residual of a method that must generalize rather than copy;
\S\ref{sec:discussion} reads it together with the Axis-B residuals.

\paragraph{A note on token cost.}
Memory's effect on token cost is model-dependent, an observation that cuts against the
claim we expected to make. On non-reasoning \code{claude-haiku-4-5}, retrieved
exemplars add a small input cost (mean 95$\rightarrow$161 tokens per call, $+66$). On a
\emph{reasoning} model (Qwen3-32B, from the free-tier runs of \S\ref{sec:setup}), the
same few-shot grounding suppresses the verbose chain-of-thought the model otherwise
emits when unsure, so net cost falls (385$\rightarrow$217 per call, $-44\%$).

\subsection{Axis B: self-knowledge}

Docs memory lifts overall accuracy from 43\% to 76\% (35$\rightarrow$62 of 82), but the
two question types behave very differently. Retrieval turns a plausible guess into a
grounded fact:

\begin{footnotesize}
\begin{verbatim}
k02 "How do I save the current file?"
 none: "Use the Save action."
       [would run: Save]              (acts, no key)
 docs: "Press C-x C-s to save
        the current buffer."          (correct)
\end{verbatim}
\end{footnotesize}

\noindent The reconfigure slice rises from 17\% to 93\% (5$\rightarrow$28 of 30).
Without docs the model invents plausible config locations: \code{r04} ``use a different
model'' points at a \code{\char`\~/.mars/config.json} that does not exist. With the
retrieved knob descriptions, which carry the exact name, file, and default, the agent
answers \code{theme\_accent} / \code{MARS\_LLM\_MODEL} in \code{tuning.json} correctly.
The two failure families that motivated the corpus design separate cleanly here, which
is how we apportion the caveat in Table~\ref{tab:axisB}: \emph{acts-instead-of-explains}
(answering a how-to question with a bare directive, as in \code{k02} above) is repaired
by the explain-don't-act instruction, while \emph{hallucinated identifiers} (a
right-idea knob under the wrong filename) are repaired only by the retrieved,
path-carrying knob descriptions. Knowledge questions gain far less (58\% to 65\%)
because they turn on an \emph{exact keybinding}, and even with the docs retrieved the
model sometimes prefers a plausible chord over the one the docs state (\code{k05}
``detach my session'' $\rightarrow$ \code{C-x C-c}, gold \code{C-x C-d});
\S\ref{sec:discussion} analyzes this knob-versus-chord asymmetry. A residual limit is
judge slack: a few correct-but-differently-phrased answers are marked wrong, so the
reported knowledge number is a mild lower bound even under the stronger
\code{claude-sonnet-5} oracle.
